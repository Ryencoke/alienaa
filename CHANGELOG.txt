ECONOMY EXPLOITS (HIGH)
[datacore.php]
- CRITICAL EXPLOIT FIX: Negative slider values in the pts[] array reduced the Drive
  cost (array_sum included negatives) while the apply loop skipped them — skills could
  be maxed for ~5 Drive. Negatives now clamped before summing; Drive deduction is
  atomic (WHERE cycles >= cost + rowCount check).
[stockex.php]
- EXPLOIT FIX: Concurrent sells of the same holding both passed the stale share check
  and both paid out (credits minted from nothing). Share decrement is now atomic
  (shares = shares - ? ... AND shares >= ?) with rowCount verification.
- Buy quantity capped at 1,000,000 to prevent integer-overflow pricing.
[bonds.php]
- EXPLOIT FIX: Two simultaneous collects of one bond both paid out. Status flip now
  happens first with AND status='active' guard; payout only if exactly 1 row changed.
  Same fix for early withdrawal.
[apartments.php]
- EXPLOIT FIX: Sell-back always refunded 50% of CATALOG price — buying a Spire Suite
  from an alt for 1 cr and selling back for 37,500 minted credits. New paid_price
  column (auto-migrated) records what was actually paid; refund = 50% of
  min(catalog, paid). Legacy rows treated as catalog. Shard-bought market units
  record 0 credit basis.
- RACE FIX: market_buy now claims the listing first (AND on_market=1 AND seller match,
  rowCount-checked) inside the transaction so two buyers can't both pay for one unit.

- EXPLOIT FIX: POSTing item_id=0 with free-text item_name created a listing backed by

  inventory item; deduction is atomic.
- RACE FIX: Bids now lock the listing row (SELECT ... FOR UPDATE). Previously two
  concurrent bids could refund the same old bidder twice and strand the other
  bidder's escrowed credits.
- BUG FIX: Cancelling a listing destroyed the listed item — it was deducted at create
  and never returned. Cancel now returns item_qty to inventory, runs in a transaction,
  and the status flip is guarded (no double-penalty on double-submit).
- Removed dead no-op query line in the bid handler.
[generalstore.php] Purchase deduction now atomic (AND creds_pocket >= price).
[welfare.php] Double-claim race fixed: the day-marker upsert is now the atomic gate

  concurrent borrows can no longer exceed the cap.
[trade.php] Direct transfer wrapped in a transaction; self-transfer check fixed
