NEW SYSTEM — THE BOUNTY BOARD (player-funded PvP contracts)
(bounty_engine.php, pages/bounties.php, lib.php, pvp.php, profile.php)
------------------------------------------------------
The player-driven counterpart to BATCH-79's PvE Contract Board: put credits
on a rival's head. Posting ESCROWS the amount from your pocket immediately
(like a trade escrow); whoever BEATS that ghost in the Arena collects every
standing bounty on them. Gives PvP a reason beyond loot/XP — grudges,
protection rackets, coordinated hunts — and a live target economy.

ECONOMY-SAFE BY CONSTRUCTION: bounties only ever MOVE credits (escrow in →
payout out), never mint. On a qualifying beat, the killer collects the sum
of everyone else's bounties on the target; bounties the killer posted
themselves are refunded to them; every active bounty on the target closes.
The harness proves the zero-sum invariant (payout + refund == total escrow)
across a 500-pool fuzz — so even collusion just shuffles a player's own
credits, no dupe.

FLOW: post (bounty_validate: no self/staff/banned targets, min 100 / max 5M,
must cover the escrow) → atomic pocket deduct + insert. Cancel anytime for a
full refund (FOR UPDATE + status gate, no double-refund). Collect is wired
into pvp.php's Arena settle on a WIN via lib.php bounty_settle_on_beat():
its own transaction, loads the target's active bounties FOR UPDATE, pays the
killer, closes them, and notifies the funders. Two attackers beating the
same mark race on the row lock, so only the first collects — no double
payout. New bounties table (lazy ensure).

SURFACED: a Bounty Board page (nav + city Combat Zone) with a post form
(handle autocomplete via players_search), your cancelable postings, a
"bounties on YOU" warning, and a marketplace of open contracts ranked by
pool with a Hunt link straight to the Arena (respecting attackability). The
Arena result shows a "Bounty Collected" chip; profiles show a bounty badge
so hunters spot marks.

HARNESS (bounty_test.php): validation rules, payout/refund partition (own
vs others), full-pool closure on a beat, and the 500-pool zero-sum fuzz.
ALL PASS. Escrow + form + hook (no new canvas), low browser-verification
risk; pages pass php -l + node --check.

--- SESSION: 2026-07-02 BATCH-78 ---

SCHEDULER/CRON PASS + DEPLOY FIX
(index.php, NEW cron.php, config.sample.php, .cpanel.yml)
------------------------------------------------------
Audited every time-based system for what actually needs a scheduler.
Finding: the game is already lazy-accrual almost everywhere — the daily
reset (index.php), bank + loan interest (ledger), Drive regen (state_api),
territory tithe/fortification (guilds), apartment expiry, bond maturity,
stock drift, auction resolution all self-heal on next access. That's the
right design for shared hosting with no guaranteed cron; very little truly
needs one. Three real issues surfaced and are fixed:

1) DEPLOY MANIFEST WAS STALE (.cpanel.yml) — the biggest one. It copied a
   hardcoded root-file list that predated most of the codebase: it omitted
   every *_engine.php (combat/net/forge/territory/transit) and several
   *_api.php endpoints, so a cPanel deploy would FATAL the pages that
   require_once them. Now copies *.php + *.js + style.css + .user.ini +
   favicon.svg (config.php is gitignored, so the server's real one is never
   overwritten). This is what would have broken the last several batches
   the moment they were deployed.

2) DAILY-RESET DOUBLE-APPLY RACE (index.php, deferred from BATCH-69) — the
   lazy per-player reset was check-then-act (SELECT marker, then reset if
   not today), so two concurrent loads at the midnight rollover could both
   grant +250 Drive and both decay skills. Now an ATOMIC claim: INSERT
   IGNORE the marker row, then UPDATE ... WHERE v<>today; only the load
   whose UPDATE affects 1 row applies the reset. Exactly one wins.

3) NO CRON ENTRY POINT despite the synopsis claiming "runs via cron" — added
   cron.php: token-guarded (CRON_TOKEN in config, hash_equals), idempotent
   bulk daily reset that only touches players whose marker isn't already
   today, so it's safe to run hourly. Its value over pure-lazy: OFFLINE
   players get reset too (fresh Signal/Health makes them valid PvP targets)
   instead of only updating when they next log in. Shares the daily_reset:{id}
   marker with the lazy path and the admin trigger, so no path can double-
   reset. config.sample.php gains CRON_TOKEN (default '' = cron.php disabled).

NEW SYSTEM — SYNDICATE TERRITORY WAR
(territory_engine.php, guilds.php, city.php, schema_territory.sql)
------------------------------------------------------
The first thing syndicates can fight each OTHER over. Eight contestable
Sprawl districts can each be held by one syndicate; a new "Territory" tab
on the Syndicate page shows the map, and city.php shows a read-only
control board. This reuses the BATCH-72 combat engine for the assaults —
the promised "turf contests on the combat engine."

THE LOOP:
- HOLD: each district you control passively accrues a credit TITHE into a
  pot and regenerates its FORTIFICATION (0-100) over real time (lazy tick,
  no cron). The Vault Keeper collects the tithe into the guild bank.
- ASSAULT: any member attacks a rival/unclaimed district by fighting its
  GARRISON — a full interactive telegraph/stamina fight (combat_engine)
  whose HP/ATK/DEF scale with the district's fortification (Sentinel
  temperament; it turtles). Costs 1 Signal + Health, like an Arena fight.
  Each win knocks fortification down (a dominant win, little HP lost, hits
  harder); when it reaches zero, control FLIPS to the attacker's syndicate
  and the old owner's uncollected tithe is lost.
- REINFORCE: the Vault Keeper spends the guild bank to add fortification
  (40 cr/point, +20 per action, up to 100).
Balance (harness-tuned): a low/freshly-claimed district falls to one
strong raider, but a fully-fortified fortress is a coordinated siege
(~36% solo win rate, ~5 clean wins to break) — passive regen + reinforcing
race against attackers, so holding contested ground takes real effort.

ENGINE + PROOF (territory_engine.php, pure logic): garrison scaling,
assault damage (margin-scaled), control-flip rules, fortification regen
(caps, no overshoot), credit yield (fort-scaled, capped), and reinforce
cost/clamps — all headless-tested, PLUS a full assault simulation driving
combat_engine against garrisons at every fortification level (win rates
100/100/100/81/36% at fort 20/40/60/80/100). ALL PASS.

SECURITY/INTEGRITY: assault is server-authoritative (client sends only
strike/guard/feint/burst/stim/flee); Signal charged atomically up front;
fortification/pot ticked under SELECT...FOR UPDATE inside the settle
transaction (no double-apply across concurrent assaults); settle-once
semantics; you can't assault your own district; reinforce/collect gated to
manage_bank rank and blocked when the syndicate is locked. New
syndicate_territory + territory_log tables (lazy ensure + schema_territory.sql).
Assaults resume across reloads. Page passes php -l + node --check; not
browser-verified (standing rule).

NEW — FABRICATION LAB FORGE MINIGAME + GEAR QUALITY
(forge_engine.php, weaponcraft.php, lib.php, player_gear.quality)
------------------------------------------------------
Crafting at the Fabrication Lab was a single click. It's now a heat-and-
strike minigame that rolls a QUALITY grade on the gear you build — the
game's first quality-differentiated equipment.

THE GAME (forge_engine.php, pure logic, headless-tested): the billet has a
temperature; you must land 5 hammer blows, and each blow scores on how
close the temp sits to the sweet band when it lands. Every action is a
DECISION, not a reflex (no timing bars) — temp, the sweet band, and the
cooling per action are all shown:
  STOKE  — bellows, temperature up
  STRIKE — land a blow (scores, consumes a strike, cools the billet)
  DRAW   — pull it back, temperature down a little
Strike too cold and it cracks; too hot and it scorches — both tank that
blow. You have a 16-action budget; run out before 5 blows land and the
unstruck blows score zero. Average blow score maps to quality 80-115%
(Flawed → Masterwork). FABRICATION SKILL widens the sweet band and raises
the ceiling (up to 125% at Fab 10), so skilled crafters roll better gear.

ECONOMY: gear stats are stored ALREADY SCALED by quality/100 (a Masterwork
+65 blueprint becomes +75 ATK; a Flawed one +52), plus a `quality` label
column — so combat, PvP, the Sim, the guild armoury, and every stat reader
are quality-aware with ZERO reader changes. Well-forged gear is now premium
goods for the guild armoury and trades. Fabrication finally has a visible,
economically-meaningful payoff.

FLOW: crafting is a three-phase forge session (forge_start reserves the ore
atomically and opens the session; forge_act runs one heat/strike round; the
final blow commits the quality-scaled gear). Abandoning refunds the ore;
the session resumes across reloads. Same server-authoritative session/AJAX
pattern as sim/thenet — the client only sends stoke/strike/draw, all scoring
is server-side. player_gear gains a `quality INT DEFAULT 100` column
(idempotent ALTER in ensure_player_gear_table); the Arsenal shows a grade
badge. The Blacksmith stays a straight purchase (bought gear = Standard
100%) — it sells finished gear rather than forging it.

HARNESS (forge_test.php): scoring-band boundaries, the clean-forge and
ran-out-of-budget paths, temp clamps, grade bands, a 3000-forge fuzz
(always terminates in budget, quality always in [80, ceiling]), and
strategy bots — heat-managing play averages 115/120/125 quality at Fab
0/5/10 vs 98/101/104 for blind mashing (a 15-20pt skill gap that never
lets mashing reach Masterwork). ALL PASS. Page passes php -l + node
--check; not browser-verified.

(sim.php, NEW combat_engine.php)
------------------------------------------------------
Fights used to auto-resolve in one POST (pick a stance radio, read a log
table). Rebuilt as real turn-based combat: each round the enemy TELEGRAPHS
its next move and the player answers — rock-paper-scissors with
information plus a stamina economy.
- ACTIONS: Strike (15 stam; interrupts charges) · Guard (+25 stam regen;
  reading a HEAVY with guard STAGGERS the enemy for a free round) · Feint
  (10 stam; OPENS a guarding enemy — next hit ×1.5) · Burst (35 stam;
  huge, brutal on staggered/charging enemies, badly punished by guard) ·
  Stim (consumes a Patch Kit mid-fight, +25 construct HP, max 2/fight,
  leaves you exposed that round — gives Foundry kits real demand) · Flee
  (eat a parting shot, keep nothing).
- ENEMIES: four temperaments (Brawler/Sentinel/Stalker/Juggernaut, hashed
  per enemy code so each drone keeps a learnable personality, shown on its
  roster card), plus CHARGE → UNLEASH (×2.5 — interrupt it or guard it).
  Telegraphs are mostly honest: higher tiers bluff more, Combat skill
  sharpens the read (55–95%, shown in the UI).

  simulation): integrity_max is 15 and NOTHING in the game raises it, so
  the old system was secretly binary — DEF >= enemy ATK meant chip-proof

  Real integrity damage on exit = ceil(taken/5), same relative-with-floor
  application as before. New soft mitigation (damage never below 25% of
  attack) kills the chip-proof degenerate case at max gear too.
- TIERS 5-7 RETUNED (sim.php's own seeds; idempotent UPDATE migration for

- Everything else preserved: Drive cost/gating, tier Combat-skill gates,

  combat_log history (now also records 'draw'/'fled'), heal panel, roster,

OVERHAUL — TRANSIT CARGO RUNS REBUILT AS "THE SWITCHYARD"
(transit.php, NEW transit_engine.php)
------------------------------------------------------
The 3-checkpoint pick-a-button cargo run (itself an upgrade over the
original single click) is replaced with a real interactive game in the
mining.php mold — server-authoritative session state, one validated AJAX
action per move, canvas client.

THE GAME: a top-down, fully-visible 13x11 container yard. You pilot a
cargo loader from the depot (bottom-left) to a district terminal
(top-right), scooping freight crates on the way, while patrol drones
sweep fixed ping-pong beats — every move YOU make, each drone takes one
step (their beats and next step are drawn on screen, so it plays as
deterministic timing/planning, not hidden dice — deliberately the
opposite identity to The Sump's fog-of-war crawler). Getting caught
loses all carried cargo and costs Health (8 + 2/crate, cap 16).
- PUSH-YOUR-LUCK WEIGHT: 7 crates per yard in three tiers (Standard
  20-32 cr / Sealed 40-60 / Volatile 75-105). Every crate carried adds
  weight; at the overload threshold drones move TWICE per step. Bigger
  hauls pay more and are genuinely harder to walk out.
- TOOLKIT: Hold (stand still, let the beats tick past — free), Dash
  (2 tiles, 3 Drive; a drone on the lane blocks it rather than killing
  you), Jammer (5 Drive, once per run, freezes every drone for 3+ moves).
- SKILL HOOKS: Scavenging +2%/lv crate value (cap +20%), Drone Ops +1
  overload threshold per 3 levels (4 base, cap 6), Netrunning +1 jam
  turn per level (cap +3). Run entry costs 6 Drive.
